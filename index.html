Nicksen Nature Audit - Accessibility

A fellow bootcamp graduate recently asked me to review her personal website for accessibility issues. This was my first time auditing a site hosted on Weebly, which has built-in backend code that can sometimes interfere with assistive technologies, one example being a screen reader that repeated certain sections of the page. After some research, I found this to be a known limitation with some website builders. In addition to the screen reader issue, I identified several other common accessibility barriers, including missing form labels and alt text, low color contrast, and improper heading structure. While each of these issues is relatively easy to fix, they are essential to ensuring that all users, regardless of ability, can navigate and interact with the site effectively.
